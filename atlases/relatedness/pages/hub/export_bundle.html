Export bundle. Single-click snapshot of every cohort-wide result this atlas can compute without picking specific dyads / triads / focal individuals. Generates 7 sections in one multi-section TSV (or one JSON blob): cohort overview, per-individual summary, per-candidate summary (with HWE + Mendelian Test A), per-family summary, full inversion-priority ranking, marker-panel design for top-N priority candidates, focal × meiosis scan for the same top-N. Per-page TSV exports remain available for analyses that do need user choices (Mendelian dyad/triad picks, Compatibility focal, BDMI scope, Regime focal).
Sections
WHAT'S IN THE BUNDLE
  1. cohort_overview · one row · cohort meta + scale + sex / status counts + call rate
  2. per_individual · one row per sample · sex, family, role, ancestry, burden, call rate, triad role
  3. per_candidate · one row per inversion · karyotype counts, HWE, hub share, Mendelian Test A, parental-meiosis counts, marker readiness
  4. per_family · one row per hub · members, sex composition, triads, within-hub edges
  5. inversion_priority · full runPriorityScan output · priority_score + SHIP/HOLD/DROP bucket
  6. marker_designs · top-N priority candidates × all tested chromosomes × interval triplets · expected_DCO + status
  7. focal_meiosis_scans · top-N priority candidates × every tested chromosome · ΔC + p_perm + status
Sections 5–7 use the meiosis stack, which is GATED on a real ngsTracts CO-call adapter (see _handoff_docs/AUDIT_2026-05-15_meiosis_stack.md). Numbers there are synthetic until the adapter ships — the bundle marks this in its # meta: header.
NOT IN THE BUNDLE (USE PER-PAGE TSV EXPORTS INSTEAD)
  • Mendelian dyad / triad tests — picks specific samples
  • Compatibility search — picks a focal individual + target karyotype
  • BDMI screen detail rows — needs scope + toggles
  • Regime inheritance — picks a focal candidate + chromosome
  • Inv × meiosis scan (full) — picks a focal candidate + scope (the bundle uses defaults for the top-N only)
